# Plugin Pattern: AI Text Generation

Use this when plugin output must be generated by `llm_client` at runtime.

## Core Rules
- If user asks for generated content, call `llm_client.chat(...)` in handler.
- Keep system prompt short and deterministic.
- Provide fallback text if model returns empty output.

## Call Pattern
```python
resp = await llm_client.chat(
    messages=[
        {"role": "system", "content": "Write one short, clean response."},
        {"role": "user", "content": user_prompt},
    ]
)
text = (resp.get("message") or {}).get("content", "").strip() or "No output generated."
```

## Return Pattern
```python
return action_success(
    facts={"text": text},
    say_hint=text,
)
```

## Guardrails
- Do not hardcode static canned outputs when generation was requested.
- Do not include long chain-of-thought text in outputs.
- Keep generated content bounded (length, style, scope).
